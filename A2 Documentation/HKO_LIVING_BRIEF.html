01 — Overview

System Overview

HKO ESL Generator is a local Python pipeline that produces 12 SENCE-accredited, CEFR aligned interactive HTML5 lessons for Chilean HR professionals, packaged as SCORM 2004 zips for Moodle upload.

What It Does

  • Reads lesson config (JSON) for all 12 lessons
  • Calls Groq (Llama-3.3-70b) to generate 14-stage lesson content
  • Generates TTS audio via edge-tts (Microsoft Neural)
  • Fetches 2 workplace images per lesson from Pexels
  • Assembles interactive HTML5 SCORM 2004 package
  • Outputs .zip + preview .html per lesson

What Each Lesson Contains

  • Hero image + can-do introduction
  • 14 interactive stages (7 vocabulary + 7 grammar)
  • 10 activity types (MCQ, match, fill, sort, flashcard, T/F, sequence, word-select, error, write)
  • 20-question final quiz (10 vocab + 10 grammar)
  • Vocabulary glossary with audio pronunciation
  • Grammar box with table, examples, common mistake
  • Full audio transcript + suggested answers

Compliance

  • CEFR A2 — Elementary level, Chilean professionals
  • SENCE Franquicia Tributaria compatible
  • SCORM 2004 4th Edition — Moodle 3.9+
  • Pass threshold: 60% quiz score
  • Progress tracked: cmi.progress_measure, cmi.score.scaled
  • Bilingual instructions (EN primary, ES supplementary)
  • No cognates vocabulary (No-Cognates Rule)

03 — Research

Scaffolding Theory — What the Evidence Says

Three frameworks converge on one non-negotiable rule: cognitive escalation from recognition to free production must never be inverted. The HKO 14-stage sequence is biologically correct.

PPP

Presentation · Practice · Production

Most structured framework. Controls cognitive sequence step by step. Ideal for A1–B1 grammar accuracy. HKO uses PPP logic inside each stage — support is withdrawn progressively until free production.

ESA

Engage · Study · Activate

Jeremy Harmer (1998). Flexible Lego-brick ordering. Emphasises student-led grammar discovery (not teacher presentation). Drives HKO hero image activation (Engage) → noticing activities (Study) → writing (Activate).

Input Hypothesis

Krashen — i+1

Acquisition requires comprehensible input at one step above current level. S3 (V) and S10 (G) are the dedicated input stages — no major output required. Forcing output here creates extraneous cognitive load and reduces encoding quality.

📐
The Absolute Rule — Never Break

Recognition → Meaning → Input study → Comprehension → Controlled production → Guided production → Free production. This is the biological sequence of L2 acquisition. Inverting or skipping any step reduces retention measurably.

05 — Research

Gamification — Evidence-Based Decisions

Research (n=139 LMS studies, MDPI 2025; Li et al. 2023 meta-analysis g≈0.82) is broadly positive with critical caveats. Design choices matter more than the presence of game elements.

✓ Implemented — Evidence Strong

  • Progress bar — rated #1 element in LMS gamification research. Visible at all times in sticky topbar.
  • XP per activity — satisfies SDT competence need. 15–40 XP per correct answer, 40 XP per stage complete.
  • Badge milestones — unlock at 200 (Word Explorer), 400 (Grammar Star), 560 (Lesson Champion) XP. Toast popup notification.
  • Immediate feedback — correct/incorrect on every activity. No delayed feedback. Most impactful single UX decision.
  • Retry without penalty — practice stages (S5, S6, S9, S12, S13) allow reset and retry. Only quiz penalises.
  • Named creative stages — 14 stages with creative names (not "Stage 1") increase engagement vs numbered metalanguage.

✗ Excluded — Risk Outweighs Benefit

  • Leaderboards — 52.7% of gamified LMSs use them but evidence shows demotivation for lower performers. Solo online A2 learner context makes this actively harmful.
  • Time pressure on core activities — timer display present but not as a hard gate. Forcing time-limited answers on vocabulary/grammar tasks increases anxiety and reduces encoding quality at A2.
  • Points for page-views — "gaming the system" risk. XP only awarded for correct answers and stage completion, not for opening a stage.
🔬
Why XP works at A2 — Self-Determination Theory (Deci & Ryan, 1985)

SDT identifies three core needs: competence, autonomy, relatedness. A2 learners have low confidence — they need frequent competence signals. XP earned per correct activity provides small, frequent "I did something right" moments. The progress bar makes competence visible across the full lesson arc. This is why the progress bar consistently outperforms badges alone in LMS research.

06 — Research

Imagery — Dual Coding and Cognitive Load

Images are a second cognitive channel, not decoration. Paivio's Dual Coding Theory (1971) + Mayer's Multimedia Learning (2009) give precise rules for when images help and when they hurt.

🧠
Dual Coding — Two Memory Traces

Vocabulary learned with a directly relevant image creates two memory traces (verbal + imagery systems). If one fades, the other cues retrieval. Concrete words + matched image = significantly stronger recall than text alone. Effect is strongest at lower proficiency levels — exactly A2.

⚠️
Cognitive Load Warning

Irrelevant or poorly matched images increase cognitive load rather than reducing it. A2 learners must allocate more resources to process visual information. Every image must have a defined pedagogical function — if it cannot be described in one sentence of purpose, remove it.

Image Slot Assignments

  • img1 (S1 hero) — Scene activation + vocabulary discovery context + S5 labelling anchor. Must be 16:9, Chilean workplace.
  • img2 (S10 grammar) — Grammar structure in visual context. Shown in grammar box. Must show target form in use.
  • Pexels queries — 2 queries per lesson pre-set in config. Landscape orientation enforced.
  • Contiguity rule — Text + image must be on same screen. Split-attention across screens eliminates the dual coding benefit.

11 — Content Standards

CS Rules — Non-Negotiable

These rules apply to every lesson, every stage, every activity. They are implemented in both the Groq system prompt and the generator HTML template.

  • 01
    NO COGNATES
    Every vocabulary word must be unrecognisable to a Spanish speaker by form alone. Test: would a Chilean professional guess this word from its spelling? If yes — replace it. Drives genuine encoding vs transparent guessing.
  • 02
    14 STAGES — SEQUENCE FIXED
    S1–S7 vocabulary half + S8–S14 grammar half. Cognitive order: recognition → input → controlled production → guided production → free production. Never inverted. Never skipped.
  • 03
    NO METALANGUAGE TO LEARNER
    Stage tags show "Vocabulary" / "Useful Language" — never "Stage 1", "Game 1", "Grammar Section", "RACD", or any system-level language. Stage names are creative English titles linked to the lesson topic.
  • 04
    S3 AND S10 ARE INPUT-ONLY
    These two stages have no major output requirement. S3 = vocabulary in authentic context (flashcards + light word_select). S10 = grammar box table study (flashcards + match). All other 12 stages require a measurable learner output.
  • 05
    S7 AND S14 = FREE WRITING ONLY
    25-40 words typed. No speaking prompts — learner is online and alone. Optional recording button (Moodle integration placeholder) must never be the only output. Always include: sentence frame + word bank with Spanish + suggested answer dropdown + rubric (Label: description format).
  • 06
    BILINGUAL INSTRUCTIONS — ENGLISH PRIMARY
    Every stage instruction: short A2 English (max 5 words per sentence) + Spanish translation in .translation-box below. Spanish is supplementary — reduces cognitive load from task decoding so attention goes to language learning.
  • 07
    WORD BANKS — ALL PRODUCTION STAGES — SPANISH INCLUDED
    Fill, sort, write activities at S5, S6, S7, S12, S13, S14 must show a word bank. A2 requirement: Spanish translation beside each word in the bank. Bank must be visible on the same screen as the task — no scrolling to find it.
  • 08
    ACTIVITY ROTATION — MAX 2× PER TYPE PER HALF
    Groq tracks type usage explicitly ("mcq: used 1x in V half") and bans any type used twice in the same half. No type appears more than twice in S1–S7 or twice in S8–S14. Violation causes attention collapse.
  • 09
    DIALOGUE — MINIMUM 1 PER LESSON
    2 speakers minimum, 6+ turns, max 12 words per line (A2 limit). Placed primarily at S4/S6 (vocab) and S11/S13 (grammar). Must be relevant to Chilean HR workplace context.
  • 10
    GRAMMAR BOX — TABLE FORMAT ONLY
    FORM table: Subject | Positive (+) | Negative (−) | Question (?). USE: ≤20 plain English words + Spanish. EXAMPLES: 3 sentences EN + Spanish. COMMON MISTAKE: ✗ wrong → ✓ correct + rule explanation. No bullet lists, no prose grammar explanations.
  • 11
    CAN-DO STATEMENTS — ENGLISH, MEASURABLE, CEFR BASED
    4 skills: escuchar / leer / escribir / identificar. Uses "podrás…" — specific and achievable at A2. No metalanguage. Appears in hero, overview card, and can-do self-check at end.
  • 12
    SPACED REVIEW — S1 WARM-UP
    S1 must include one activity using 3 vocabulary words preparing the student for the lesson context. Words passed via prev_vocab parameter. For Lesson 1: uses words from within Lesson 1's own list. Ensures no lesson is an island.
  • 13
    AUDIO — MAX 12 WORDS PER SENTENCE (A2)
    Scripts in config are pre-written to A2 sentence length. TTS via edge-tts Microsoft Neural, rate -8%. Dialogue audio: speaker labels prepended ("María says: …"). S4 and S11 have audio play button + inline transcript toggle.
  • 14
    IMAGES — FUNCTIONAL, MOTIVATIONAL, CONTEXT GIVING ROLES
    img1: S0 hero, S1 + S10 grammar context. img3: S10 grammar image. Pexels queries pre-set per lesson (Chilean workplace, landscape 16:9). No decorative images. Every image has a defined pedagogical function.

Present Perfect

function buildActivity(act, uid) {
  const type = act.type || 'mcq';
  const heading = act.heading || '';
  const d = act.data || act; // support both {type,data:{...}} and flat structures

  switch(type) {
    case 'mcq':    return renderMCQ(uid, heading, d);
    case 'match':  return renderMatch(uid, heading, d);
    case 'fill':   return renderFill(uid, heading, d);
    case 'sort':   return renderSort(uid, heading, d);
    case 'flashcard': return renderFlashcard(uid, heading, d);
    case 'truefalse': return renderTF(uid, heading, d);
    case 'error':  return renderError(uid, heading, d);
    case 'word_select': return renderWordSelect(uid, heading, d);
    case 'sequence': return renderSequence(uid, heading, d);
    case 'write':  return renderWrite(uid, heading, d);
    default:       return renderMCQ(uid, heading, d);
  }
}

// ── MATCH ─────────────────────────────────────────────────
function pickMatch(uid, side, idx) {
  const st = matchState[uid];
  if (!st || document.getElementById(`${uid}_L${idx}`)?.classList.contains('matched')) return;
  if (!st.sel) {
    st.sel = { side, idx };
    document.getElementById(`${uid}_${side}${idx}`)?.classList.add('selected');
  } else {
    const prev = st.sel;
    if (prev.side === side) {
      document.getElementById(`${uid}_${prev.side}${prev.idx}`)?.classList.remove('selected');
      st.sel = { side, idx };
      document.getElementById(`${uid}_${side}${idx}`)?.classList.add('selected');
      return;
    }
    const li = prev.side === 'L' ? prev.idx : idx;
    const ri = prev.side === 'R' ? prev.idx : idx;
    const match = li === ri;
    ['L','R'].forEach(s => {
      const el = document.getElementById(`${uid}_${s}${s==='L'?li:ri}`);
      if (el) {
        el.classList.remove('selected');
        el.classList.add(match ? 'matched' : 'wrong');
        if (!match) setTimeout(() => el.classList.remove('wrong'), 700);
      }
    });
    if (match) { addXP(10); st.done++; if (st.done === st.total) { const fb=document.getElementById(`${uid}_fb`); if(fb){fb.style.display='block';fb.className='fb ok';fb.textContent='✅ All matched! Great work.';} } }
    st.sel = null;
  }
}

let activeBlank = null;
function selectBlank(bl) {
  document.querySelectorAll('.blank.active-blank').forEach(b => b.classList.remove('active-blank'));
  if (activeBlank === bl) { activeBlank = null; return; } // toggle off
  activeBlank = bl;

function fillBlank(uid, pill, word) {
  let target = null;
  // Use selected blank if it belongs to this uid
  if (activeBlank && activeBlank.id && activeBlank.id.startsWith(uid + '_bl')) {
    target = activeBlank;
    activeBlank = null;
  } else {
    // Fall back: fill first empty blank
    document.querySelectorAll(`[id^="${uid}_bl"]`).forEach(bl => {
      if (!target && bl.textContent.trim() === '_____') target = bl;
    });
  }
  if (!target || target.textContent.trim() !== '_____') return;
  target.textContent = word;
  target.dataset.filled = word;
  target.classList.remove('active-blank');
  pill.classList.add('used');
}
function checkFill(uid, count) {
  activeBlank = null;
  let correct = 0;
  for (let i = 0; i < count; i++) {
    const bl = document.getElementById(`${uid}_bl${i}`);

    const ans = bl.dataset.answer || '';
    if (bl.textContent.trim().toLowerCase() === ans.toLowerCase()) {
      bl.classList.add('ok'); correct++;
    } else if (bl.textContent.trim() !== '_____') {
      bl.classList.add('err');
    }
  }
  const fb = document.getElementById(`${uid}_fb`);
  if (fb) {
    fb.style.display = 'block';
    if (correct === count) { fb.className='fb ok'; fb.textContent=`✅ All correct! +25 XP`; addXP(25); }
    else { fb.className='fb warn'; fb.textContent=`${correct} of ${count} correct. Check the highlighted gaps.`; }

}

function moveSeq(uid, btn) {
  const item = btn.closest('[data-orig]');
  const list = document.getElementById(`${uid}_seqlist`);
  if (!item || !list) return;
  const dir = btn.dataset.dir === 'up' ? -1 : 1;
  const items = [...list.children];
  const pos = items.indexOf(item);
  const newPos = pos + dir;
  if (newPos < 0 || newPos >= items.length) return;
  if (dir === -1) list.insertBefore(item, items[pos - 1]);
  else list.insertBefore(items[pos + 1], item);
}
function checkSeq(uid, count) {
  const container = document.getElementById(`${uid}_seq`);
  const list = document.getElementById(`${uid}_seqlist`);
  if (!container || !list) return;
  let correct_order;
  try { correct_order = JSON.parse(container.dataset.correct); } catch(e) { return; }
  const items = [...list.children];
  let correct = 0;
  items.forEach((item, pos) => {
    const origIdx = parseInt(item.dataset.orig);
    if (correct_order[pos] === origIdx) { item.classList.add('correct'); correct++; }
    else item.classList.remove('correct');
  });
  const fb = document.getElementById(`${uid}_fb`);
  if (fb) {
    fb.style.display = 'block';
    if (correct === count) { fb.className='fb ok'; fb.textContent='✅ Perfect order! +30 XP'; addXP(30); }
    else { fb.className='fb warn'; fb.textContent=`${correct} of ${count} in the right position.`; }
  }
}
